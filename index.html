Módulos del schema — hacé clic para ver el código de cada uno
1
Tenant + User + Role
Multi-tenant base. Refresh token en User. UNIQUE(tenantId, email).
2
Product + Category + Brand
Catálogo. Categorías con árbol recursivo (parentId). UNIQUE(tenantId, code).
3
StockMovement + Deposit
Tabla inmutable. Stock = SUM de movimientos. Costo promedio ponderado incremental.
4
Document + DocumentItem
Factura o Remito directo, no ambos. PuntoDeVenta AFIP múltiple. Estado: DRAFT → CONFIRMED → CANCELLED.
5
Customer + CUIT + CurrentAccount
UNIQUE parcial WHERE cuit IS NOT NULL (raw migration). CC como entries, saldo = SUM derivado.
6
PriceList + Payment + Supplier
Listas de precio por cliente/grupo. Pagos multi-método. Proveedores con CUIT.
1
Crear el monorepo
mkdir pintureria-erp && cd pintureria-erp
pnpm init
pnpm add -D turbo
mkdir -p apps/api apps/web packages/db
2
Inicializar cada app
# NestJS
cd apps/api && npx @nestjs/cli new . --skip-git

# Next.js
cd ../web && npx create-next-app@latest . \
  --typescript --tailwind --app --skip-git

# Prisma package
cd ../../packages/db
pnpm init && pnpm add prisma @prisma/client
npx prisma init
3
Crear base de datos en Neon
Entrás a neon.tech → New Project → copiás la connection string. La ponés en .env
DATABASE_URL="postgresql://user:pass@ep-xxx.neon.tech/erp?sslmode=require"
4
Primera migración
cd packages/db
npx prisma migrate dev --name init
npx prisma generate
5
Habilitar Row-Level Security
-- En Neon SQL Editor:
ALTER TABLE "User" ENABLE ROW LEVEL SECURITY;
ALTER TABLE "Product" ENABLE ROW LEVEL SECURITY;
-- (repetir para cada tabla del tenant)

CREATE POLICY tenant_isolation ON "Product"
  USING (tenant_id = current_setting('app.tenant_id'));
1
Deploy NestJS → Fly.io
npm install -g flyctl
cd apps/api
fly auth login
fly launch          # detecta Dockerfile, elige región sa (São Paulo)
fly secrets set DATABASE_URL="..." JWT_SECRET="..."
fly deploy
2
Deploy Next.js → Vercel
cd apps/web
npx vercel          # login + link repo
# En Vercel dashboard: Settings → Environment Variables
# NEXT_PUBLIC_API_URL = https://tu-app.fly.dev
3
Dominio Namecheap → Vercel
En Namecheap → Advanced DNS → agregar CNAME: www → cname.vercel-dns.com. En Vercel → Settings → Domains → agregar tu dominio.
4
Seeder: primer tenant + usuario
